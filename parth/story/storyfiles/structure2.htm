Story File Structure 2

On this page, we will give more information about the use of a story file to create a story.

The path-to-a-story-file must point to a story file in the same web domain as the Story Utility html page.

The reason for this restriction is that the story file is loaded using Javascript and the tool XMLHttpRequest. Therefore standard browser security restrictions apply.

We said that each line in the story file that starts with a non-blank character should consist of:

  • The url for a slide in the story
  • Whitespace
  • The text of a title to identify the slide

The url for an individual slide may be quite arbitrary because a slide is loaded into an HTML iframe. There are no browser security restrictions in specifying the source for an iframe.

For the url, you may use:

  • A relative url where relative is interpreted as relative to the web directory of the story file
  • A general url that opens with http:// or https://

In a few cases such as for the New York Times web site, the target page is designed so that it will not permit itself to be displayed in an iframe. For such a page, the Story Utility may not be used.
