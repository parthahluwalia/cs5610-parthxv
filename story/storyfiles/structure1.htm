Story File Structure 1

The Story Utility is normally called with a query string that points to a story file. A story file provides a list of the links to each web page that will form a slide in the story.

The path-to-a-story-file should be provided as a url that is placed in the query string when the Story Utility is invoked. 

..../story/index.htm?path-to-a-story-file

By default, the story opens with slide 1. By using an additional query parameter start, you may specify the number N of the slide to start with. 

..../story/index.htm?path-to-a-story-file&start=N

Normally, N should fall between 1 and the total number of slides. However, if N is outside of this range, then via “automatic wraparound” the value will be mapped into the proper range. For example, the setting start=0 will cause the story to open with the last slide.

A story file should be structured as follows.

Each line in the story file that starts with a non-blank character should consist of the data for one slide in the story:

  • The url for a slide in the story
  • Whitespace
  • The text of a title to identify the slide

The title may contain whitespace internally.

If the title is omitted then the url will become its own title. Usually, it is far better to supply an explicit title for each slide.

Each line in the story file that is empty or starts with a blank is ignored when constructing the slide show. This enables spacing and comments in the story file.

If the path-to-a-story-file points to a web directory but omits the story file name then the story file name is assumed to be story.txt. This convention makes it easier to reference a story.
